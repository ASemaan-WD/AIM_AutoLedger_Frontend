## Key Interactions

### Export Flow (Matched â†’ Reviewed)
1. User clicks **Export** button
2. If form is dirty, automatically saves changes
3. Calls `onSendForApproval` callback
4. Backend updates status to "reviewed"
5. UI updates to show QueuedIndicator
6. Action buttons disappear

### Error Recovery Flow (Error â†’ ...)
1. User sees ErrorAlert with details
2. Options:
   - Click **Delete** to remove invoice
   - Click **Need Help** to contact support
3. Support resolves issue
4. Backend updates status (typically back to Pending or Matched)
5. UI updates to show appropriate state

### View in AIM Vision (Exported)
1. User clicks **View in AIM Vision** button
2. Opens external link to ERP system
3. Invoice displayed in read-only mode in ERP

## Accessibility Considerations

- All interactive elements have proper `aria-label` attributes
- Color is not the only indicator (icons + text used)
- Keyboard navigation supported for all buttons
- Focus states clearly visible
- Screen reader friendly status announcements

## Responsive Behavior

- Panel maintains fixed width (320px) on all screen sizes
- Indicators stack vertically on narrow panels
- Button groups wrap appropriately
- Text truncates with ellipsis where needed

## Animation Details

- Spinner: Continuous rotation animation (360deg, 1s linear infinite)
- State transitions: Instant (no fade/slide animations)
- Button hover: Smooth color transition (150ms ease)
- Focus rings: Instant appearance for accessibility

## Notes

- Line item navigation is **always enabled** regardless of state (per requirements)
- Export button combines Save + Mark as Reviewed in one action
- Balance alert only shows for Matched state with non-zero balance
- CompletenessChecker is hidden for Pending state (PendingStateIndicator shown instead)
- Error state uses generic "Need Help" button that opens email (can be customized)
